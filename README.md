# happy_save

## 행복을 저금하는 다이어리

### 프론트 스택

React / TypeScript

### 백엔드 스택

Spring

### 서비스 방향

일단 웹 -> PWA 적용

### 브랜치 관리

main -> develop 에서 frontend / backend 두 브랜치로 나뉨
각자 frontend / backend 브랜치에서 생성한 이슈대로 (태스크대로) 하위 브랜치 생성
작업 후 각 frontend / backend 브랜치로 병합
frontend / backend 브랜치에서 develop 브랜치로 병합
develop 브랜치에서 main 브랜치로 병합
(병합 시마다 교차확인하면 좋음)
